Sybil-Resistance & Bot-Prevention Architecture

Phase 1: Beta Active Phase 2: Production Coming Soon

üìã Executive Summary

AEra implements a two-phase security model that combines cryptographic proof, social validation, and blockchain transparency to prevent fake accounts, spam, and bot attacks while maintaining a frictionless user experience.

Current Status: Phase 1 (Market Entry - Frictionless Onboarding)

Future: Phase 2 (Community Gate - Resonance Score Threshold)

üéØ Core Security Principles

1. Blockchain as Single Source of Truth

  • All critical data lives on-chain (Identity NFTs, Resonance Scores, Interactions)
  • Database serves only as performance cache
  • Transparent, immutable, publicly auditable

2. Cryptographic Proof of Ownership

  • Every action requires MetaMask signature (personal_sign)
  • No passwords, no KYC, no personal data
  • Wallet = Identity = Proof of Human

3. Social Validation Network

  • Resonance Score = Own Activity + Follower Validation
  • Bots have high scores but zero social proof
  • Community self-regulates through transparency

üìä Two-Phase Security Model

üöÄ Phase 1: Market Entry (Current - Beta)

Goal: Frictionless onboarding for early adopters

‚úÖ Active Security Features:

  1. MetaMask Signature Requirement
    • Every login requires active signature
    • Challenge-Response with unique nonce
    • Cannot be bypassed or automated
  2. Identity NFT (Gasless Mint)
    • Backend pays gas fees
    • User only signs transaction
    • Soul-bound token (non-transferable)
  3. Score Growth Rate-Limiting
    • Maximum +1 point per 60 seconds
    • Cap at 100 points maximum
  4. Self-Follow Prevention
    • Users cannot follow themselves
    • Prevents self-boosting attacks
  5. Blockchain Transaction Recording
    • All interactions logged on-chain
    • Publicly verifiable via BaseScan
    • Immutable audit trail

‚ö™ Intentionally Open (For Beta Testing):

  • No NFT Score-Gating - Anyone can mint Identity NFT
  • No Resonance Threshold - All users can access private communities
  • Purpose: Allow first users to experience the full flow

üîê Phase 2: Community Gate (Future - Production)

Goal: Sybil-resistant community access via Resonance Score

üîê Enhanced Security Features:

1. Resonance Score Threshold

MIN_RESONANCE = 110  # Threshold for gate access

# Gate Check Algorithm
def check_gate_access(address):
    # Step 1: Verify Identity NFT (on-chain)
    has_nft = web3_service.has_identity_nft(address)
    
    if not has_nft:
        return "Access Denied: No Identity NFT"

        return f"Access Denied: Score {total_resonance}/110 required"

total_resonance = own_score + avg_follower_score

# avg_follower_score = SUM(follower_scores) / follower_count

# Example Scenarios:

3. Social Proof Mechanism

No artificial rate limits needed! The Resonance Score itself acts as the natural gate:

  • Quality over Quantity: High-scoring followers boost resonance more
  • Organic Growth: Real communities naturally accumulate engaged followers
  • Bot Prevention: Fake followers (score 50-60) provide minimal boost
  • Transparency: Blockchain reveals suspicious patterns

üìä Example: 100 real followers (avg score 65) = 165 resonance vs. 1000 bots (avg score 50) = 150 resonance

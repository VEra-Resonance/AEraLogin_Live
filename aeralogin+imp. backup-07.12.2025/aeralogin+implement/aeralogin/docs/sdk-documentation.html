üöÄ Quick Start

The simplest setup requires only a few lines of code. The SDK automatically protects all elements with the data-aera-protect attribute.

üí° Attribute Names

Both data-aera-protect and data-aera-protected are supported. Use whichever fits your coding style better.

HTML
<!-- Public Content --> <div>This content is visible to everyone.</div> <!-- Protected Content --> <div data-aera-protect> Only visible to NFT holders! </div> <!-- Protected with minimum score --> <div data-aera-protect data-min-score="50"> Requires Score ‚â• 50 </div> <!-- Load and initialize SDK --> <script src="https://aeralogin.com/sdk/aera-gate.js"></script> <script> AEraGate.init({ clientId: 'your-client-id', autoProtect: true }); </script>

‚öôÔ∏è AEraGate.init(options)

Initializes the SDK with the specified options.

Parameters

Option Type Default Description
clientId Required string - Your Client ID from the AEra Dashboard
autoProtect boolean true Automatically protect elements with data-aera-protected
mode string 'modal' Login mode: 'modal' or 'redirect'
theme string 'dark' UI theme: 'dark' or 'light'
requireNFT boolean true NFT ownership required
minScore number 0 Minimum Resonance Score
onAuthenticated function - Callback after successful authentication
onLogout function - Callback after logout
onError function - Callback on errors

Example

JavaScript
AEraGate.init({ clientId: 'my-website-client', autoProtect: true, mode: 'modal', theme: 'dark', minScore: 50, onAuthenticated: function(user) { console.log('Welcome!', user.wallet); console.log('Score:', user.score); }, onLogout: function() { console.log('Goodbye!'); }, onError: function(error) { console.error('Error:', error); } });

üì± Register Your Application

Before using the OAuth flow, you need to register your application to get a client_id and client_secret.

‚úÖ Prerequisites
  • An AEra Identity NFT (mint one at AEraCore Dashboard)
  • Your website URL and callback URL ready

Option 1: Via Dashboard (Recommended)

1. Go to AEraCore Dashboard
2. Connect your wallet and sign in
3. Navigate to "SDK Apps" section
4. Click "Register New App" and fill in the details
5. Copy and securely store your client_id and client_secret

Option 2: Via API

Register programmatically using a JWT token from dashboard login:

HTTP Request
POST https://aeralogin.com/api/oauth/my-apps/register Authorization: Bearer <your_dashboard_jwt_token> Content-Type: application/json { "app_name": "My Awesome Website", "website_url": "https://mywebsite.com", "redirect_uri": "https://mywebsite.com/callback", "description": "My protected web application" }

Response

JSON
{ "success": true, "client_id": "aera_1a2b3c4d5e6f7g8h...", "client_secret": "AbCdEfGhIjKlMnOpQrStUvWxYz...", "message": "‚ö†Ô∏è Save your client_secret NOW! It cannot be retrieved later." }
‚ö†Ô∏è Security Notice
  • The client_secret is shown ONLY ONCE at registration
  • Store it securely (e.g., environment variables, secrets manager)
  • NEVER commit secrets to version control
  • If compromised, regenerate via the dashboard

üîë OAuth 2.0 Flow (Server-Side)

For server-side verification, use the OAuth 2.0 Authorization Code flow. This provides maximum security as the token exchange happens on your server with your client_secret (never exposed to the browser).

‚ö†Ô∏è Important: Server-Side Required

The client_secret must NEVER be used in frontend code! The Token Exchange (Step 2) must happen on your backend server.

1. Authorization URL (Frontend)

Redirect the user to the AEraLogIn authorization page:

URL Parameters
https://aeralogin.com/oauth/authorize ?client_id=YOUR_CLIENT_ID &redirect_uri=https://yoursite.com/callback &response_type=code &state=RANDOM_STATE_FOR_CSRF_PROTECTION
Parameter Required Description
client_id Required Your OAuth Client ID from the AEra Dashboard
redirect_uri Required Must match exactly one of your registered redirect URIs
response_type Required Always code
state Recommended Random string for CSRF protection. Returned unchanged.

2. Token Exchange (Server-Side) ‚ö†Ô∏è

After successful authentication, the user is redirected to your redirect_uri with a code parameter. Your server must exchange this for an Access Token:

Node.js / Express
// Express.js callback route app.get('/callback', async (req, res) => { const { code, state } = req.query; // 1. Validate state parameter (CSRF protection) if (state !== req.session.oauthState) { return res.status(403).send('Invalid state parameter'); } // 2. Exchange code for access token const tokenResponse = await fetch('https://aeralogin.com/oauth/token', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ grant_type: 'authorization_code', code: code, redirect_uri: 'https://yoursite.com/callback', client_id: process.env.AERA_CLIENT_ID, client_secret: process.env.AERA_CLIENT_SECRET // ‚ö†Ô∏è Server-side only! }) }); const tokenData = await tokenResponse.json(); // 3. Check for errors if (tokenData.error) { return res.status(400).json({ error: tokenData.error_description }); } // 4. Store session and redirect req.session.aeraToken = tokenData.access_token; req.session.wallet = tokenData.wallet; req.session.score = tokenData.score; req.session.hasNFT = tokenData.has_nft; res.redirect('/dashboard'); });
Python / Flask
import requests, os from flask import Flask, request, session, redirect @app.route('/callback') def oauth_callback(): code = request.args.get('code') state = request.args.get('state') # 1. Validate state if state != session.get('oauth_state'): return 'Invalid state', 403 # 2. 3. Token Verification (Optional Server-Side Check)

You can verify a stored token at any time using the /api/v1/verify endpoint:

Node.js - Token Verification
// Verify token on each protected request async function verifyAeraToken(accessToken) { const response = await fetch('https://aeralogin.com/api/v1/verify', { method: 'POST', headers: { 'Authorization': `Bearer ${accessToken}` } }); return await response.json(); } // Usage in middleware app.use('/protected', async (req, res, next) => { const token = req.session.aeraToken; if (!token) return res.redirect('/login'); const result = await verifyAeraToken(token); if (!result.valid) { delete req.session.aeraToken; return res.redirect('/login'); } // Check NFT requirement if (!result.has_nft) { return res.status(403).send('NFT required'); } req.user = result; next(); });

4. NFT Verification Endpoint (Alternative)

For additional security, you can verify NFT ownership directly with client credentials:

JavaScript (Node.js)
const verification = await fetch('https://aeralogin.com/api/oauth/verify-nft', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ access_token: accessToken, client_id: process.env.AERA_CLIENT_ID, client_secret: process.env.AERA_CLIENT_SECRET }) }); const result = await verification.json(); // { valid: true, wallet: "0x...", has_nft: true, score: 150, chain_id: 8453 }

Error Handling

Common OAuth errors and how to handle them:

Error Description Solution
invalid_client Unknown client or invalid secret Check your client_id and client_secret
invalid_grant Authorization code expired or already used Code is valid for 60 seconds and single-use only
unsupported_grant_type Wrong grant_type value Use authorization_code
invalid_request Missing required parameters Include all required fields in your request

üîç API Verification

Use the /api/v1/verify endpoint to verify a user's session token server-side. This is useful when you need to validate authentication state from your backend.

Request

HTTP
POST https://aeralogin.com/api/v1/verify Authorization: Bearer <aera_session_token>

Response (Success)

JSON
{ "valid": true, "authenticated": true, "wallet": "0x1234...abcd", "score": 42, "has_nft": true, "chain_id": 8453, "issued_at": "2025-01-01T00:00:00Z", "expires_at": "2025-01-02T00:00:00Z" }

Response (Failure)

JSON
{ "valid": false, "authenticated": false, "error": "Token expired" }

Example (Node.js)

JavaScript
const response = await fetch('https://aeralogin.com/api/v1/verify', { method: 'POST', headers: { 'Authorization': `Bearer ${sessionToken}` } }); const result = await response.json(); if (result.valid && result.has_nft) { // User is authenticated and has NFT console.log('User wallet:', result.wallet); } else { // Authentication failed console.log('Error:', result.error); }

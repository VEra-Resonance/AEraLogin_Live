AEra Gate SDK

The AEra Gate SDK enables website operators to protect their content with NFT-based access control. Users must own an AEra Identity NFT to access protected areas.

‚ú® Benefits
  • No custom login system required
  • Decentralized authentication via wallet
  • Automatic protection of HTML elements
  • Easy integration in 5 minutes

üîß Installation

Via CDN (Recommended)

HTML
<script src="https://aeralogin.com/sdk/aera-gate.js"></script>

Via NPM

Terminal
npm install @aera/gate-sdk
JavaScript
import { AEraGate } from '@aera/gate-sdk';

üöÄ Quick Start

The simplest setup requires only a few lines of code. The SDK automatically protects all elements with the data-aera-protected attribute.

HTML
<!-- Public Content --> <div>This content is visible to everyone.</div> <!-- Protected Content --> <div data-aera-protected> Only visible to NFT holders! </div> <!-- Load and initialize SDK --> <script src="https://aeralogin.com/sdk/aera-gate.js"></script> <script> AEraGate.init({ clientId: 'your-client-id', autoProtect: true }); </script>

‚öôÔ∏è AEraGate.init(options)

Initializes the SDK with the specified options.

Parameters

Option Type Default Description
clientId Required string - Your Client ID from the AEra Dashboard
autoProtect boolean true Automatically protect elements with data-aera-protected
mode string 'modal' Login mode: 'modal' or 'redirect'
theme string 'dark' UI theme: 'dark' or 'light'
requireNFT boolean true NFT ownership required
minScore number 0 Minimum Resonance Score
onAuthenticated function - Callback after successful authentication
onLogout function - Callback after logout
onError function - Callback on errors

Example

JavaScript
AEraGate.init({ clientId: 'my-website-client', autoProtect: true, mode: 'modal', theme: 'dark', minScore: 50, onAuthenticated: function(user) { console.log('Welcome!', user.wallet); console.log('Score:', user.score); }, onLogout: function() { console.log('Goodbye!'); }, onError: function(error) { console.error('Error:', error); } });

üë§ AEraGate.getUser()

üì° Events

Das SDK l√∂st Events aus, auf die Sie reagieren k√∂nnen. Verwenden Sie AEraGate.on() um Event-Listener zu registrieren.

Verf√ºgbare Events

Event Beschreibung Callback Parameter
login Benutzer hat sich erfolgreich angemeldet { wallet, score, hasNFT }
logout Benutzer hat sich abgemeldet { wallet }
sessionExpired Session Token ist abgelaufen { wallet, expiredAt }
error Ein Fehler ist aufgetreten { code, message }

Beispiel

JavaScript
// Login-Event abfangen AEraGate.on('login', (user) => { console.log('User logged in:', user.wallet); console.log('Resonance Score:', user.score); }); // Session-Ablauf behandeln AEraGate.on('sessionExpired', (data) => { console.log('Session expired, please login again'); // Optionaler Re-Login AEraGate.login(); }); // Fehlerbehandlung AEraGate.on('error', (error) => { console.error('AEra Error:', error.message); });

üîë OAuth Flow (Server-Side)

For server-side verification, use the OAuth 2.0 flow. This provides maximum security as verification takes place on your server.

1. Autorisierungs-URL

URL
https://aeralogin.com/oauth/authorize ?client_id=YOUR_CLIENT_ID &redirect_uri=https://yoursite.com/callback &response_type=code &state=RANDOM_STATE

2. Token Exchange

JavaScript (Node.js)
const response = await fetch('https://aeralogin.com/oauth/token', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ grant_type: 'authorization_code', code: code, redirect_uri: 'https://yoursite.com/callback', client_id: 'YOUR_CLIENT_ID', client_secret: 'YOUR_CLIENT_SECRET' }) }); const { access_token } = await response.json();

3. NFT Verification

JavaScript (Node.js)
const verification = await fetch('https://aeralogin.com/api/oauth/verify-nft', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ access_token: access_token, client_id: 'YOUR_CLIENT_ID', client_secret: 'YOUR_CLIENT_SECRET' }) }); const result = await verification.json(); // { valid: true, wallet: "0x...", has_nft: true, score: 150, chain_id: 8453 }

üîç API Verification

Use the /api/v1/verify endpoint to verify a user's session token server-side. This is useful when you need to validate authentication state from your backend.

Request

HTTP
POST https://aeralogin.com/api/v1/verify Authorization: Bearer <aera_session_token>

Response (Success)

JSON
{ "valid": true, "authenticated": true, "wallet": "0x1234...abcd", "score": 42, "has_nft": true, "chain_id": 8453, "issued_at": "2025-01-01T00:00:00Z", "expires_at": "2025-01-02T00:00:00Z" }

Response (Failure)

JSON
{ "valid": false, "authenticated": false, "error": "Token expired" }

Example (Node.js)

JavaScript
const response = await fetch('https://aeralogin.com/api/v1/verify', { method: 'POST', headers: { 'Authorization': `Bearer ${sessionToken}` } }); const result = await response.json(); if (result.valid && result.has_nft) { // User is authenticated and has NFT console.log('User wallet:', result.wallet); } else { // Authentication failed console.log('Error:', result.error); }
